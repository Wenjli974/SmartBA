# 🤖 SmartBA - 智能商业分析助手

> 基于 Vue 3 和 OpenAI 的智能商业分析助手，让需求分析和文档生成变得简单高效。

## 📋 项目概述

SmartBA 是一个基于 Vue 3 和 OpenAI 的商业分析助手，它能够帮助用户：
- 📝 智能分析和检查项目需求
- 🎯 自动生成用户故事和验收标准
- 📊 在线管理和编辑需求内容
- 📄 生成专业的项目需求文档

系统提供两种工作模式：**Agent模式**和**基础模式**，满足不同场景下的业务分析需求。

## 💼 系统工作模式

### 🧠 Agent模式 - 智能代理系统

Agent模式下，SmartBA 通过智能代理系统提供更强大的自主分析能力：

#### 🧩 MasterAgent 主代理
`MasterAgent.ts` 是整个代理系统的核心控制器，负责：

- 🔄 协调和调度多个分析工具
- 📊 跟踪并报告分析进度
- 💭 执行"思考"过程并记录代理的决策思路
- 📝 评估项目需求完整性并提出改进建议
- 🧩 为不同类型的分析创建执行计划

#### 🛠️ 智能分析工具 (Tools)

位于 `src/agents/tools` 目录下，包含多个专用分析工具：

- **BusinessAnalysisTool**: 分析项目背景，自动生成用户故事，并检查需求细节问题
- **FunctionalAnalysisTool**: 拆解功能需求，生成用户故事和测试用例
- **UIUXAnalysisTool**: 分析用户界面需求，提供界面设计建议和原型参考

这些工具通过 `ToolRegistry` 进行统一注册和管理，便于 MasterAgent 调用。

#### 📦 上下文管理 (Memory)

位于 `src/agents/memory` 目录下：

- **ContextManager**: 管理项目分析的上下文信息
  - 存储和检索项目背景和分析结果
  - 在本地存储中持久化上下文数据
  - 为 AI 调用生成上下文摘要

#### 📊 Agent模式分析流程

1. 用户输入项目背景信息
2. MasterAgent 评估需求完整性
3. 系统自动提出补充问题（如需要）
4. 创建针对项目的分析计划
5. 按优先级执行各分析工具
6. 整合分析结果，生成完整的需求文档

### 🔄 基础模式 - 交互式分析

基础模式下，SmartBA 提供直观的交互式界面，让用户可以手动控制分析过程：

#### 主要功能模块

基础模式包含三个主要功能模块（标签页）：

##### 1. 业务分析 (Business Analysis)
- 📊 项目背景分析和需求检查
- 🔍 智能识别需求缺失点
- 💡 交互式需求完善
- 📋 生成业务需求文档

##### 2. 功能分析 (Function Analysis)
- 📝 用户故事生成和管理
- ✅ 验收标准及测试用例自动生成
- 📊 表格化展示和编辑
- 📄 需求文档一键生成

##### 3. UI/UX 分析
- 🎨 界面原型建议
- 📱 用户体验分析
- 🔄 交互流程设计
- 💻 界面布局建议

#### 基础模式优势

- 👤 用户主导分析流程
- 👁️ 直观的可视化界面
- ✏️ 灵活的编辑功能
- 🔍 细粒度的分析控制

## 📈 分析类型
 
无论使用哪种模式，SmartBA 都支持三种核心分析类型：
- **业务分析 (Business Analysis)**: 分析项目背景，识别关键业务流程
- **功能分析 (Functional Analysis)**: 拆解功能需求，生成用户故事和测试用例
- **UI/UX 分析**: 分析用户界面需求，提供界面设计建议

## 🚀 快速开始

### 环境要求
- Node.js 16+
- npm 8+

### 安装步骤

1. 克隆项目
```bash
git clone [项目地址]
cd SmartBA
```

2. 安装依赖
```bash
npm install
```

3. 配置环境变量
```bash
cp .env.example .env
```
编辑 .env 文件，设置您的 OpenAI API Key：
```
VITE_OPENAI_API_KEY=your_api_key_here
```

4. 启动开发服务器
```bash
npm run dev
```

## 📖 使用指南

### 选择工作模式
在应用首页，您可以选择使用：
- **Agent模式**：更自动化、流程化的分析体验
- **基础模式**：更直观、可控的手动分析体验

### Agent模式使用流程
1. 选择Agent模式
2. 输入项目背景描述
3. 系统自动评估需求完整性并提出问题
4. 回答系统提出的问题（如有）
5. 查看分析进度和中间思考过程
6. 获取综合分析结果

### 基础模式使用流程

#### 1. 业务分析模块
1. 进入业务分析页面
2. 输入项目背景描述
3. 使用"需求检查"功能完善需求
4. 点击"AI生成"生成业务分析文档
5. 查看和导出生成的文档

#### 2. 功能分析模块
1. 进入功能分析页面
2. 输入项目背景描述
3. 使用"AI生成"生成用户故事
4. 在表格中编辑和管理用户故事
5. 点击"生成需求文档"按钮生成详细文档
6. 导出用户故事或需求文档

#### 3. UI/UX 分析模块
1. 进入 UI/UX 分析页面
2. 输入界面需求描述
3. 获取界面设计建议和原型参考
4. 查看交互流程建议
5. 导出分析结果

### 数据管理
- 所有模块的数据都会自动保存
- 可以随时导出数据备份
- 支持跨模块数据关联
- 提供多种导出格式（Excel、Markdown等）

## ⚙️ 项目结构

```
src/
├── components/        # 组件目录
├── views/            # 页面视图
├── services/         # 服务层（API 调用等）
├── assets/           # 静态资源
├── agents/           # 智能代理系统
│   ├── MasterAgent.ts         # 主代理控制器
│   ├── tools/                # 分析工具集
│   │   ├── ToolRegistry.ts   # 工具注册机制
│   │   ├── BusinessAnalysisTool.ts # 业务分析工具
│   │   ├── FunctionalAnalysisTool.ts # 功能分析工具
│   │   └── UIUXAnalysisTool.ts # UI/UX分析工具
│   └── memory/               # 上下文管理
│       └── ContextManager.ts  # 项目上下文管理
└── App.vue          # 根组件
```

## ⚠️ 注意事项

### API 配置
- 需要配置有效的 OpenAI API Key

- 用户故事数据保存在本地存储中

### 使用建议
- 建议先使用需求检查功能
- 确保项目背景描述完整
- 及时保存或导出重要内容
